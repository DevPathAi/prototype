# 테스트 계획: LCS 우선 가이드 학습 루프

생성: 2026-06-19 `/plan-eng-review`  
브랜치: `main`  
저장소: `DevPathAi/prototype`

## 영향받는 페이지와 경로

- `/prototype/` 개요: LCS 우선 히어로, 기본 동작 `3분 데모 시작`, 보조 플랫폼 콘텐츠를 검증합니다.
- `/prototype/` 데모 구역: 가이드 루프가 기본이며 진단, 경로, 샌드박스, 검색이 보조 기능 탐색으로 남아 있는지 검증합니다.
- `/prototype/` 아키텍처/기술 구역: 데모를 떠났다가 돌아올 때 가이드 세션 상태만 보존되는지 검증합니다.

## 검증할 핵심 상호작용

- 개요 히어로에서 가이드 데모를 시작하고 1단계에 도착합니다.
- 1280px와 390px 뷰포트에서 다섯 단계를 순서대로 완료합니다.
- 데스크톱 2열과 모바일 세로 배치에서 일반 답변과 맥락 기반 답변을 비교합니다.
- `currentContent`, `currentPath`, `activeTags`, `recentErrors`를 토글하고 `recentErrors`가 기본 제외 상태인지 확인합니다.
- 모든 맥락 필드를 끄면 맥락 기반 리뷰, 경로 차이, 증거, 요약이 설명과 함께 사용 불가 상태가 되는지 확인합니다.
- 완료된 단계로 돌아가 맥락을 변경하면 이후 파생 결과가 원자적으로 초기화되는지 확인합니다.
- 미래 단계를 시도했을 때 상태 변경 없이 선행 조건 안내가 표시되는지 확인합니다.
- 완료 동작을 중복 실행해도 하나의 전환만 적용되고 오류 UI가 나오지 않는지 확인합니다.
- 다른 최상위 구역으로 이동했다가 돌아왔을 때 마지막 유효 단계에서 계속할 수 있는지 확인합니다.
- 진행 후 재시작을 한 번 확인하면 모든 상태와 초점이 원자적으로 초기화되는지 확인합니다.
- 보이는 멘토 요약을 복사하고 미리보기/클립보드 내용이 정확히 같은지 확인합니다.
- 클립보드 성공, 권한 실패, 지연된 완료, 재시작 이후 오래된 완료를 주입해 검증합니다.
- 진단 수집기를 주입하고 이벤트가 단계, 동작, 이름 붙은 오류, 시도 ID만 포함하는지 확인합니다.
- 누락, 모순, 민감 테스트 데이터를 트리거했을 때 재시도 불가능한 안전 오류 패널과 기능 탐색 탈출 경로가 표시되는지 확인합니다.

## 경계 사례

- 필드가 하나도 없는 상태를 포함한 LCS 필드 16개 조합 전체
- 비밀값처럼 보이는 토큰, 이메일 주소, 절대 경로, 개인 식별값, 원본 오류 로그 테스트 데이터 값
- `Applied`, `Rejected(duplicateAction)`, `Rejected(prerequisiteMissing)`, `Failed(invalidLoopState)` reducer 결과
- 복사 시도가 재시작 이후, 다른 복사 시도 이후, 데모 구역 이탈 이후 완료되는 경우
- 390px 너비, 1280px 너비, 확대된 글자 배율, 키보드 전용 내비게이션, 보이는 초점, 실시간 영역 안내, 움직임 줄이기
- 공유 위젯 추출 이후 기존 모바일 서비스 헤더, 선택된 데모 페이지 의미 정보, 실시간 샌드박스 상태, API/저장소/모드 상태 표시
- 페이지 새로고침은 새 세션을 시작하고, 일반 구역 내비게이션은 새 세션을 시작하지 않는 경우
- `main.dart.js`가 +5% 경고 예산 또는 +10% 실패 예산을 초과하는 경우
- 배포 후 Pages 루트, `main.dart.js`, 기준 경로가 404가 아닌 응답을 반환하는지

## 핵심 경로

1. `3분 데모 시작` -> 실패 -> 일반 AI -> LCS 선택 -> 맥락 기반 리뷰 -> 경로 차이 -> 취업 준비 증거 -> 멘토 요약
2. LCS 필드 변경 -> 하위 결과 무효화 -> 다시 생성된 결과가 승인된 맥락만 포함
3. 복사 요청 -> 주입된 플랫폼 부수 효과 -> 일치하는 시도 완료 -> 보이는 상태; 오래된 완료는 상태를 바꾸지 않음
4. 잘못된 테스트 데이터 -> 카탈로그 검증 실패 -> 안전한 재시도 불가 오류 -> 기존 기능 탐색은 계속 사용 가능
5. 공유 위젯 추출 -> 모든 최상위 구역과 기능 시뮬레이터에 계속 도달 가능하고 반응형 유지
6. 분석 -> 커버리지 포함 테스트/골든 -> 릴리스 빌드 -> 크기 예산 -> Pages 배포 -> HTTP 스모크 -> 390px/1280px 수동 완료

## 커버리지 게이트

- `lib/learning_loop/`: 100% 라인 커버리지
- Reducer: 명시적 전환 행렬과 최소 1,000개의 결정적 혼합 동작 시퀀스
- 선택자: 비활성 값 부재 검증을 포함한 16개 개인정보 조합
- Goldens: 390px와 1280px 주요 단계의 Ubuntu 기준 이미지
- 기존 동작: LCS 동작 변경 전 기존 회귀 테스트가 먼저 통과해야 함
