# Test Plan: LCS-First Guided Learning Loop

Generated by `/plan-eng-review` on 2026-06-19  
Branch: `main`  
Repo: `DevPathAi/prototype`

## Affected Pages/Routes

- `/prototype/` overview: verify the LCS-first hero, one primary `3분 데모 시작` action, and secondary platform content.
- `/prototype/` demo section: verify the guided loop is primary and diagnosis, path, sandbox, and search remain reachable as secondary feature exploration.
- `/prototype/` architecture and technology sections: verify navigation away from and back to the demo preserves only the guided-session state.

## Key Interactions to Verify

- Start the guided demo from the overview hero and land on stage 1.
- Complete all five stages in order on 1280px and 390px viewports.
- Compare generic and contextual responses in desktop two-column and mobile stacked layouts.
- Toggle `currentContent`, `currentPath`, `activeTags`, and `recentErrors`; verify `recentErrors` starts excluded.
- Disable all context fields and verify contextual review, path delta, evidence, and summary become unavailable with an explanation.
- Return to a completed stage, change context, and verify all dependent later output clears atomically.
- Attempt a future stage and verify prerequisite guidance without state mutation.
- Double-activate completion and verify only one transition applies without error UI.
- Navigate to another top-level section and return through `계속하기` at the last valid stage.
- Restart after progress, confirm once, and verify all state and focus reset atomically.
- Copy the visible mentor summary and verify exact preview/payload equality.
- Inject clipboard success, permission failure, delayed completion, and stale completion after restart.
- Inject a collecting diagnostic sink and verify events contain only stage, action, named error, and attempt ID.
- Trigger missing, contradictory, and sensitive fixtures and verify the non-retryable safe error panel plus feature-exploration escape route.

## Edge Cases

- All 16 LCS field combinations, including the zero-field state.
- Secret-like token, email address, absolute path, personal identifier, and raw error-log fixture values.
- `Applied`, `Rejected(duplicateAction)`, `Rejected(prerequisiteMissing)`, and `Failed(invalidLoopState)` reducer outcomes.
- Copy attempt completes after restart, after another copy attempt, and after leaving the demo section.
- 390px width, 1280px width, increased text scaling, keyboard-only navigation, visible focus, live-region announcements, and reduced motion.
- Existing mobile service header, selected demo-page semantics, live sandbox status, and API/storage/mode status pills after shared-widget extraction.
- Page refresh starts a new session; ordinary section navigation does not.
- `main.dart.js` exceeds the +5% warning and +10% failure budgets.
- Pages root, `main.dart.js`, and base path return non-404 responses after deployment.

## Critical Paths

1. `3분 데모 시작` -> failure -> generic AI -> LCS selection -> contextual review -> path delta -> employment evidence -> mentor summary.
2. LCS field change -> downstream invalidation -> regenerated outputs contain only approved context.
3. Copy request -> injected platform effect -> matching attempt completion -> visible status; stale completion never changes state.
4. Invalid fixture -> catalog validation failure -> safe non-retryable error -> existing feature exploration remains usable.
5. Shared-widget extraction -> every existing top-level section and feature simulator remains reachable and responsive.
6. Analyze -> test with coverage/goldens -> release build -> size budget -> Pages deploy -> HTTP smoke -> manual 390px/1280px completion.

## Coverage Gates

- `lib/learning_loop/`: 100% line coverage.

- Selectors: all 16 privacy combinations with absence assertions for disabled values.
- Goldens: Ubuntu-canonical baselines for key stages at 390px and 1280px.
- Existing behavior: regression tests land before any LCS behavior change.
