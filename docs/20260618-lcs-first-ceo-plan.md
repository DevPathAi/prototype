# CEO 계획: LCS 우선 가이드 학습 루프

생성: 2026-06-18 `/plan-ceo-review`  
브랜치: `main` | 모드: 선택적 확장  
저장소: `DevPathAi/prototype`

## 비전

프로토타입을 독립 기능 시뮬레이터 모음에서 하나의 가이드 증명으로 전환합니다. DevPath는 학습자가 왜 실패했는지 이해하고, 학습자가 승인한 맥락만 사용하며, AI 리뷰 품질을 바꾸고, 다음 학습 과제를 조정하고, 취업 준비 또는 멘토 후속 상담에 쓸 수 있는 증거를 남겨야 합니다.

3분 경험은 별도 내레이션 없이 다음 다섯 질문에 답해야 합니다.

1. 무엇이 실패했는가?
2. 일반 AI는 무엇을 놓쳤는가?
3. DevPath는 어떤 맥락을 사용했으며, 학습자가 그것을 통제했는가?
4. 그 맥락 때문에 무엇이 바뀌었는가?
5. 취업 준비 또는 멘토 후속 상담에 남는 유용한 산출물은 무엇인가?

## 현재 상태

- `DemoSection`은 진단, 경로, 샌드박스, 검색, 리뷰 상태를 중앙에서 들고 있지만 화면은 독립 페이지처럼 렌더링합니다.
- `_SandboxReviewSimulator`는 이미 실패 코드, 실행 출력, 정적 AI 리뷰를 제공합니다.
- `_PathGeneratorSimulator`와 `_PathExecutionBoard`는 생성된 경로와 다음 과제를 렌더링합니다.
- `_ServiceDemoPage`, 반응형 그리드, 상태 의미 정보, 모바일 내비게이션은 재사용 가능한 표현 패턴을 제공합니다.
- 기존 테스트는 독립 기능 내비게이션, 샌드박스 리뷰 결과, 아키텍처 콘텐츠, 모바일 구역 전환을 검증합니다.
- LCS는 학습자 승인 스냅샷을 수집, 정제, 조립, 노출하는 얇은 맥락 다리입니다. AI 리뷰와 경로 보정은 LCS의 책임이 아니라 LCS 스냅샷의 소비자입니다.

## 선택한 구현 접근

기존 독립 페이지에 조건부 문구를 더하는 대신, 명시적인 LCS 소비 학습 루프 모델을 도입합니다.

- `LearningLoopScenario`가 실패 과제, 일반 답변, 맥락 스냅샷, 맥락 기반 리뷰, 경로 차이, 취업 준비 증거, 멘토 요약 테스트 데이터를 소유합니다.
- `LearningLoopStage`가 가이드 흐름의 유효한 진행 단계를 정의합니다.
- `LearningContextSnapshotView`는 학습자가 승인한 맥락 필드, 출처, 최신성, 정제된 값을 포함합니다.
- AI 리뷰와 경로 보정은 스냅샷을 소비합니다. LCS는 맥락 제공자로 남습니다.
- 기존 기능 페이지는 보조 탐색 경로로 유지합니다.
- 새 루프 컴포넌트는 `lib/main.dart`에 계속 쌓지 않고 `lib/learning_loop/` 아래로 분리합니다.

## 핵심 리뷰 결정

### 상태 무효화

학습자가 뒤로 이동해 입력을 바꾸면 그 지점 이후의 모든 파생 결과를 비웁니다. AI 리뷰, 경로 차이, 취업 준비 증거, 멘토 요약은 새 승인 맥락에서 다시 생성되어야 합니다. UI는 오래된 상위 상태에서 만들어진 결과를 보여주면 안 됩니다.

### 빈 맥락

모든 LCS 필드는 선택 사항이며 비활성화될 수 있습니다. 승인된 필드가 하나도 없으면 흐름은 일반 답변 상태로 돌아가고 맥락 기반 리뷰, 경로 보정, 취업 준비 증거, 멘토 요약 생성을 비활성화합니다. UI는 학습자가 승인한 맥락이 필요하다는 설명을 표시해야 합니다.

### 취업 준비 증거

취업 준비 증거는 채용 예측, 준비도 비율, 자격 증명이 아니라 관찰 가능한 과제 개선 기록입니다. 통과한 테스트, 자료 전후 교정, 입증된 역량 태그, 다음 보강 과제를 포함합니다. 문구는 고용 보장이나 리크루터 검증을 암시하면 안 됩니다.

### 멘토 요약 개인정보 보호

복사되는 멘토 요약은 학습자가 승인한 맥락 필드와 그 필드에서 파생된 결과만 포함합니다. 비활성 필드 값과 필드 이름은 빠져야 합니다. 요약은 일부 학습 맥락이 학습자 선택으로 제외되었을 수 있다고만 말할 수 있습니다.

### 재시작 동작

초기 단계 이후 재시작은 확인이 필요합니다. 재시작은 모든 선택과 파생 결과를 함께 비우고 가이드 루프 제목으로 초점을 돌리며 접근성 실시간 영역으로 초기화를 알립니다. 부분 초기화 상태는 유효하지 않습니다.

### 반응형 가이드 내비게이션

데스크톱은 전체 단계 표시기를 보여줍니다. 모바일은 현재 단계 카드, `현재 / 전체` 진행, 기본 이전/다음 동작을 보여주며 전체 단계 목록은 펼침 컨트롤로 제공합니다. 데스크톱과 모바일은 같은 `LearningLoopStage` 상태와 전환 규칙을 소비합니다.

### 오류 상태

- `SummaryCopyFailure`: 요약을 계속 보이게 하고 실패를 알리며 재시도와 수동 텍스트 선택을 제공합니다.
- `ScenarioDataMissing`: 단언문 또는 빈 렌더링 대신 루프 본문을 재시도 불가능한 테스트 데이터 오류 패널로 대체하고 기존 기능 탐색 경로를 제공합니다.
- `TransitionResult`: `Applied`, `Rejected(reason)`, `Failed(error)`를 반환합니다. 중복 완료는 조용히 거부하고, 잠긴 단계 접근은 선행 조건을 설명하며, 불변식 실패만 복구를 노출합니다.

### 접근성 피드백

각 단계 전환은 새 단계 제목으로 초점을 이동합니다. 단계 변경, 생성된 결과, 초기화, 복사 성공, 이름 붙은 실패는 범위가 지정된 실시간 영역으로 알립니다. 키보드 순서는 화면 순서를 따르고, 모든 동작은 보이는 초점을 유지해야 합니다. 움직임 줄이기 요청 시 전환 애니메이션에 의미를 의존하지 않습니다.

## 아키텍처 결정

### 학습 루프 상태 소유권

`LearningLoopState`는 불변입니다. 순수 전환 함수가 현재 상태와 이름 붙은 동작을 받아 `Applied(nextState)`, `Rejected(reason)`, `Failed(error)`를 반환합니다. 상위 입력 변경은 같은 적용 전환 안에서 의존하는 하위 값을 비웁니다. 위젯은 상태를 렌더링하고 동작만 전달하며 로컬 콜백에 무효화 규칙을 구현하지 않습니다. 외부 상태 관리 패키지는 추가하지 않습니다.

```text
PrototypeHomePage
└── DemoSection
    ├── GuidedLearningLoop
    │   ├── LearningLoopState
    │   ├── transition(state, action)
    │   ├── LearningLoopScenario 테스트 데이터
    │   └── 단계 화면
    └── 기존 기능 탐색 페이지
```

### 모듈 경계

새 기능은 `lib/learning_loop/` 아래에 둡니다.

- `models.dart`: 불변 상태, 시나리오, 스냅샷, 증거, 이름 붙은 결과/오류 타입
- `reducer.dart`: 동작, 유효 전환, 의존성 무효화, 불변식 검사
- `selectors.dart`: 맥락 기반 리뷰, 경로 차이, 증거, 보이는 요약, 클립보드 내용 파생
- `fixtures.dart`: 정적 시나리오와 일반/맥락 기반 응답 테스트 데이터
- `view.dart`: 가이드 루프 구성과 반응형 단계 내비게이션
- `components.dart`: 데스크톱/모바일 단계 화면이 공유하는 단계별 표현 컴포넌트

`lib/theme/prototype_theme.dart`는 애플리케이션 `ThemeData`와 의미 기반 `ThemeExtension`을 소유합니다. 이 파일은 `docs/DESIGN.md`의 실행 대응물입니다. 셸이나 `lib/learning_loop/`는 두 번째 팔레트나 기능별 토큰 묶음을 정의하지 않습니다.

### 기존 페이지 격리

가이드 루프와 보조 기능 페이지는 같은 타입 시나리오 테스트 데이터를 사용해 학습자, 과제, 예상 결과 라벨 일관성을 유지합니다. 그러나 상호작용 상태는 동기화하지 않습니다. `기능 탐색` 안의 컨트롤은 진행 중인 가이드 루프를 변경하거나 무효화할 수 없습니다. 루프로 돌아오면 마지막 유효 상태를 복원합니다.

### 가이드 세션 생명주기

`_PrototypeHomePageState`가 현재 불변 `LearningLoopState`를 소유하고 `DemoSection`에 상태와 업데이트 콜백을 전달합니다. 개요, 아키텍처, 기술 구역으로 이동하면 데모 화면은 해제될 수 있지만 가이드 세션 상태는 유지됩니다. 돌아오면 마지막 유효 상태에서 화면을 재구성하고 `계속하기`를 제공합니다. 기존 진단, 경로, 샌드박스, 검색 탐색 상태는 로컬이며 독립적으로 초기화될 수 있습니다.

### 파생 결과 계약

타입 시나리오 테스트 데이터는 독립 작성된 최종 카드가 아니라 출처 사실을 담습니다. `contextualReview(state)`, `pathDelta(state)`, `evidence(state)`, `mentorSummary(state)` 같은 순수 선택자가 같은 학습자 승인 스냅샷에서 하위 결과를 파생합니다. 비활성 필드는 선택자 결과에 나타나면 안 되고, 빈 맥락은 명시적인 사용 불가 결과를 반환합니다.

### LCS 필드 계약

`LearningContextSnapshotView`는 문서 26과 맞춘 네 개의 학습자 통제 필드만 포함합니다: `currentContent`, `currentPath`, `activeTags`, 정제된 `recentErrors`. 앞의 세 필드는 기본 포함이고 `recentErrors`는 기본 제외이며 명시적 선택 포함이 필요합니다. 현재 실습 과제, 제출 코드, 테스트 결과는 활성 상호작용 입력이지 LCS 필드로 중복하지 않습니다.

### 클립보드 부수 효과 경계

`GuidedLearningLoop`는 주입 가능한 `ClipboardWriter` 콜백을 받습니다. 운영 기본값은 `Clipboard.setData`를 호출합니다. 테스트는 결정적 성공, 이름 붙은 실패, 지연된 완료 콜백을 주입합니다. 화면은 부수 효과 전에 `CopyRequested(attemptId)`를 전달하고 부수 효과 후 `CopySucceeded(attemptId)` 또는 `CopyFailed(attemptId)`를 전달합니다. reducer는 플랫폼에 의존하지 않고 오래된 시도 ID를 거부합니다.

## 오류와 복구

예상 가능한 도메인 결과는 예외가 아니라 봉인된 결과 타입으로 전달합니다. reducer는 `Applied`, `Rejected`, `Failed`를 반환하고 selector는 `Derived`, `Unavailable`, 특정 `Failure`를 반환합니다. 플랫폼 API만 기능 경계 밖에서 예외를 던질 수 있으며, `Clipboard.setData`의 `PlatformException`은 `SummaryCopyFailure`로 매핑합니다. 모든 오류를 조용히 삼키는 fallback은 금지합니다.

각 복사 요청은 시도 ID를 받습니다. 최신 요청이 진행 중인 동안 복사 동작은 비활성입니다. 성공/실패는 시도 ID가 현재 상태와 일치할 때만 상태를 갱신합니다. 재시작은 시도 ID를 무효화하므로 오래된 플랫폼 완료가 초기화된 화면을 바꿀 수 없습니다.

모든 시나리오는 `_PrototypeHomePageState` 초기화 시점에 디버그/릴리스 모두에서 한 번 검증합니다. 검증은 필수 비어 있지 않은 출처 사실, 유효한 단계 참조, 최소 하나의 선택 가능한 맥락 필드, 의미 있는 경로 차이, 선택자 호환성, 모순 증거 부재를 확인합니다. 실패는 재시도 불가능한 `ScenarioDataMissing` 패널을 만들고, 테스트 데이터 테스트는 CI에서 실패합니다.

## 보안 결정

테스트 데이터 값은 사전 정제된 출처만 허용합니다. 검증기와 테스트는 비밀값처럼 보이는 토큰, 이메일 주소, 절대 로컬 경로, 개인 식별값, 원본 오류 로그 값을 거부합니다. 런타임 마스킹은 다층 방어일 뿐이며 민감한 출처 데이터 커밋을 정당화하지 않습니다.

보이는 최종 요약 텍스트와 클립보드 내용은 같은 선택자에서 나와야 합니다. 명시적 동작 라벨은 `현재 보이는 내용 복사`입니다. 숨은 메타데이터, 비활성 필드, 내부 ID, 대체 직렬화를 추가하지 않습니다. 가이드 시나리오는 개인 이름이나 가명 대신 역할/상황 라벨을 사용합니다.

## 데이터 흐름과 상호작용

데모 구역을 떠나도 마지막 유효 가이드 루프 상태는 메모리에 보존됩니다. 돌아오면 해당 단계에서 계속할 수 있습니다. 내비게이션은 로컬 저장소, URL 매개변수, 서버에 상태를 저장하지 않습니다. 페이지 새로고침은 새 세션을 시작합니다. 명시적 재시작만 페이지 세션 안의 인메모리 루프를 비웁니다.

학습자는 현재 단계 또는 완료된 단계로 이동할 수 있습니다. 미래 단계는 비활성이며 충족되지 않은 선행 조건을 의미 정보/도움말 텍스트로 노출합니다. 완료된 단계의 데이터를 변경하면 이후 단계와 파생 결과를 무효화하고, 다시 진행하려면 새 결과를 생성해야 합니다.

모든 완료 동작은 예상 출처 단계를 포함합니다. reducer는 예상 단계가 현재 단계와 같을 때만 진행합니다. 이미 완료된 단계에서 중복 동작이 들어오면 `Rejected(duplicateAction)`을 반환하고 사용자 대상 오류 UI는 보여주지 않습니다. 잠긴 단계 접근은 `Rejected(prerequisiteMissing)`으로 선행 조건 안내를 제공합니다. 불가능한 불변식은 `Failed(invalidLoopState)`를 반환하고 마지막 유효 상태를 보존합니다.

최신성은 `이번 실습 실행`, `현재 경로 기준`, `진단 완료 시점` 같은 시나리오 상대 라벨을 사용합니다. 맥락 영역에는 보이는/의미 정보용 `정적 샘플` 라벨이 있어야 하며 현재 시각이나 실시간 데이터 조회를 암시하면 안 됩니다.

## 코드 품질

구현은 동작 의미가 다른 두 개의 커밋으로 나누는 것이 이상적입니다. 첫 커밋은 공유 프로토타입 위젯 추출이며 기존 테스트가 통과해야 합니다. 두 번째 커밋은 가이드 루프 도메인, UI, 부수 효과, 테스트, 문서, 배포 변경을 추가합니다. 구조 이동과 제품 동작을 같은 커밋에 섞지 않습니다.

`ScenarioId`, `ContextFieldId`, `LearningLoopStage`는 타입 식별자입니다. 새 기능 코드는 사용자 대상 문자열을 맵 키로 사용하지 않고 테스트 데이터 조회를 강제 언랩하지 않습니다. 누락된 식별자는 봉인된 결과 계약을 통해 `ScenarioDataMissing`으로 전달합니다.

`main.dart`의 재사용 표현 기본 요소, 즉 패널, 결과 행, 반응형 그리드, 상태 표현은 `lib/widgets/prototype_components.dart`로 이동하고 공개된 기능 중립 이름을 사용합니다. 기존 페이지와 가이드 루프는 같은 구현을 가져옵니다. 테마 구성과 의미 토큰은 `lib/theme/prototype_theme.dart`로 분리합니다.

## 성능과 배포

`main.dart.js` 기준값, 폰트 자산 예산, 전체 정적 자산 예산을 분리합니다. JavaScript와 전체 자산은 +5%에서 경고, +10%에서 실패입니다. 폰트 바이트는 기록된 상한을 초과하면 실패입니다. 릴리스 빌드는 `--base-href /prototype/`를 유지합니다.

| 배포 후 스모크 | 페이지/자산 non-2xx 또는 404 | `PagesSmokeFailure` | 검증 실패와 되돌리기 결정 | GitHub Actions 검사 실패 |

```text
PrototypeHomePage

├── OverviewSection
│   └── LCS 우선 히어로 ──▶ 가이드 데모 시작

    │   ├── 순수 전환 reducer

    └── 보조 기능 탐색
        ├── 진단
        ├── 경로
        ├── 샌드박스
        └── 검색
```

### 예외 경로를 포함한 데이터 흐름

```text
FIXTURE ─▶ 검증 ─▶ 맥락 선택 ─▶ 파생 ─▶ 렌더링/복사
   │           │              │             │           │
   ├ 누락     ├ 비밀값       ├ 비어 있음  ├ 오래됨   ├ 플랫폼 실패
   └ 잘못됨   └ 모순         └ 모두 비활성 └ 불일치   └ 오래된 시도
       │           │              │             │           │
       └────────── 이름 붙은 결과 + 보이는 복구, 조용한 실패 금지 ──────────┘
```

### 상태 머신

```text
IDLE
  └─ 시작 ─▶ 1 실패 + 일반 AI
                 └─ 완료(1) ─▶ 2 LCS 미리보기
                                      ├─ 비어 있음 ─▶ 맥락 기반 결과 사용 불가
                                      └─ 완료(2) ─▶ 3 맥락 기반 리뷰
                                                            └─ 완료(3) ─▶ 4 경로 차이
                                                                                  └─ 완료(4) ─▶ 5 증거 + 요약

완료된 단계 ── 뒤로/수정 ──▶ 해당 단계, 이후 모든 결과 비움
진행된 단계 ── 재시작 확인 ──▶ IDLE
미래 단계 ──▶ Rejected(prerequisiteMissing), 상태 보존
중복 완료 ──▶ Rejected(duplicateAction), 상태 보존
불가능한 불변식 ──▶ Failed(invalidLoopState), 상태 보존
```

```text
   ─▶ 릴리스 웹 빌드 ─▶ JavaScript/정적/폰트 크기 예산
   ─▶ Pages 산출물 업로드 ─▶ 배포 ─▶ 재시도 HTTP 스모크

```

## 오래된 다이어그램 점검

- `_ArchitectureMap`, `_ArchitectureFlow`, `_CommunicationMatrix`, `_DataDomainOwnership`는 운영 아키텍처 설명으로 유지되며 이 프로토타입 UI 계획 때문에 의미가 바뀌지 않습니다.
- 개요 `_FlowPreview`는 새 내러티브에 맞게 `진단 → 경로 → 실습 → 리뷰`에서 LCS 우선 이야기로 바뀌어야 합니다.
- README의 독립 시뮬레이터 설명은 오래되었으므로 가이드 루프가 기본이고 기능 페이지가 보조임을 문서화해야 합니다.
