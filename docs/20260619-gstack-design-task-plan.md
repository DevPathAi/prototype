## 목표

승인된 GStack 리뷰/디자인 산출물을 실행 가능한 작업 흐름으로 전환합니다. 프로토타입의 핵심 증명은 LCS 우선 가이드 학습 루프가 되고, 기존 시뮬레이터는 보조 탐색으로 유지합니다. 동시에 디자인, 개인정보 보호, 접근성, 배포 게이트를 측정 가능한 기준으로 보존합니다.

## 작업 흐름

### T1. 디자인 시스템 계약

- 우선순위: P1
- 출처: 디자인 리뷰 T1, 엔지니어링 재실행 테마 토큰 작업
- 범위:
  - `docs/DESIGN.md`를 시각 기준의 단일 출처로 유지합니다.
  - 실행 가능한 토큰은 `lib/theme/prototype_theme.dart`에 둡니다.
  - 재사용 표현 컴포넌트는 `lib/widgets/prototype_components.dart`에 둡니다.
- 승인 기준:
  - 승인된 팔레트, 8px 반경, 초점, 움직임, 반응형, 증거 규칙이 문서화됩니다.
  - 새 UI는 기능별 팔레트를 만들지 않고 공유 테마/토큰을 읽습니다.

### T2. 공유 UI 추출

- 우선순위: P1
- 출처: CEO/엔지니어링 공유 UI 작업
- 범위:
  - 기능에 종속되지 않는 패널, 그리드, 구역 헤더, 상태 행을 추출하거나 재사용합니다.
  - 가능한 범위에서 구조 리팩터링과 기능 동작 변경을 분리합니다.
- 승인 기준:
  - 기존 개요, 데모, 아키텍처, 기술 구역에 계속 도달할 수 있습니다.
  - 기존 시뮬레이터 회귀 테스트가 계속 통과합니다.

### T3. 학습 루프 도메인 카탈로그

- 우선순위: P1
- 출처: CEO/엔지니어링 도메인 카탈로그 작업
- 범위:
  - 타입이 있는 시나리오, 단계, LCS 필드, 증거, 결과 모델을 정의합니다.
  - 정적 테스트 데이터 카탈로그를 렌더링 전에 한 번 검증합니다.
  - 네 개의 LCS 필드를 강제합니다: `currentContent`, `currentPath`, `activeTags`, `recentErrors`.
- 승인 기준:
  - `recentErrors`는 기본 제외입니다.
  - 비밀값처럼 보이는 토큰, 이메일, 절대 경로, 원본/개인 테스트 데이터 값이 거부됩니다.
  - 잘못된 카탈로그는 재시도 불가능한 안전 오류 패널을 렌더링합니다.

### T4. 상태 머신과 선택자

- 우선순위: P1
- 출처: 엔지니어링 상태 머신/선택자 작업
- 범위:
  - 불변 reducer를 `Applied`, `Rejected`, `Failed` 결과와 함께 구현합니다.
  - 맥락 기반 리뷰, 경로 차이, 증거, 멘토 요약, 클립보드 내용 파생은 선택자에 둡니다.
  - 승인 맥락이 바뀌면 하위 결과를 원자적으로 비웁니다.
- 승인 기준:
  - 미래 단계는 잠긴 상태입니다.
  - 중복 완료는 상태를 손상시키지 않고 거부됩니다.
  - LCS 포함 조합 16개 모두에서 비활성 값이 모든 파생 결과에서 제외됩니다.

### T5. 부수 효과 경계

- 우선순위: P1
- 출처: CEO 클립보드 작업, 엔지니어링 부수 효과 작업
- 범위:
  - 클립보드 작성기와 진단 수집기를 주입 가능하게 만듭니다.
  - 오래된 비동기 완료를 처리하기 위해 복사 시도 ID를 추적합니다.
  - 진단은 메타데이터만 유지합니다.
- 승인 기준:
  - 복사 성공, 실패, 재시도, 오래된 완료가 테스트에서 결정적으로 검증됩니다.
  - 보이는 멘토 요약과 클립보드 내용이 정확히 일치합니다.
  - 숨은 메타데이터, 비활성 필드, 내부 ID가 복사되지 않습니다.

### T6. 가이드 UX 구현

- 우선순위: P1
- 출처: CEO/디자인/엔지니어링 가이드 UX 작업
- 범위:
  - 다섯 단계 흐름을 구현합니다.
    1. 실패 + 일반 AI
    2. LCS 승인
    3. 맥락 기반 리뷰 비교
    4. 경로 보정
    5. 증거 + 멘토 요약
  - 재시작 확인, 잠긴 단계 안내, 빈 맥락 상태, 무효화 알림, 실시간 영역 메시지, 반응형 내비게이션을 추가합니다.
- 승인 기준:
  - 심사위원은 무엇이 실패했는지, 일반 AI가 무엇을 놓쳤는지, 어떤 맥락이 승인되었는지, 무엇이 바뀌었는지, 어떤 산출물이 남았는지 답할 수 있습니다.
  - 완료 계층은 취업 준비 증거를 먼저 보여주고 멘토 요약을 두 번째로 보여줍니다.
  - 준비도 점수, 자격 증명, 채용 보장, 불안을 자극하는 취업 문구를 사용하지 않습니다.

### T7. 제품 셸 통합

- 우선순위: P2
- 출처: CEO/디자인/엔지니어링 제품 셸 작업
- 범위:
  - 개요 히어로를 증거 중심 CTA로 바꿉니다.
  - `데모 > 학습 루프`를 기본 목적지로 둡니다.
  - 진단/경로/샌드박스/검색은 보조 기능 탐색 아래에 유지합니다.
  - 가이드 세션 상태를 `DemoSection`보다 위로 끌어올립니다.
- 승인 기준:
  - 구역 내비게이션이 가이드 세션 상태를 보존합니다.
  - 페이지 새로고침은 새 세션을 시작합니다.
  - 기존 시뮬레이터는 가이드 루프 상태와 격리된 채 계속 사용할 수 있습니다.

### T8. 테스트 인프라

- 우선순위: P1
- 출처: 엔지니어링 테스트 인프라 작업, 엔지니어링 재실행 테스트 행렬 작업
- 범위:
  - `lib/learning_loop/` 기능 커버리지를 강제합니다.
  - reducer, selector, 테스트 데이터, 위젯, 반응형, 재시작, 클립보드, 회귀 테스트를 추가합니다.
  - 시각 기준 흐름이 준비되면 골든/브라우저 확대 후속 커버리지를 추가합니다.
- 승인 기준:
  - `flutter test --coverage`가 통과합니다.
  - `tool/check_feature_coverage.dart`는 100% 미만에서 실패합니다.
  - 너비 `599`, `600`, `979`, `980`이 소형/중간 폭/넓은 폭 동작을 검증합니다.
  - 빈 맥락, 이전 단계 수정에 따른 무효화, 재시작, 클립보드 실패가 테스트됩니다.

### T9. 배포 예산과 스모크

- 우선순위: P1/P2
- 출처: 엔지니어링 배포 작업, 엔지니어링 재실행 배포 예산 작업
- 범위:
  - JavaScript, 폰트, 전체 정적 자산 예산을 분리합니다.
  - 배포 이후 Pages HTTP 스모크를 추가합니다.
  - 워크플로우에 현재 릴리스 기준값을 기록합니다.
- 승인 기준:
  - `tool/check_web_size.dart`는 JS/전체 자산이 +5%를 넘으면 경고, +10%를 넘으면 실패를 냅니다.
  - 폰트 바이트는 상한을 넘을 수 없습니다.
  - Pages 루트와 `main.dart.js`가 배포 이후 404가 아닌 응답을 반환합니다.

## T10 검증 결과

실행 기준: 로컬 릴리스 산출물 `build/web`을 `http://127.0.0.1:4175/prototype/`로 제공하고 GStack 브라우저 환경의 Playwright로 검증했습니다.

- `1280x900`: `3분 데모 시작`부터 `증거/멘토 요약`까지 완료. `scrollWidth == clientWidth == 1280`, 콘솔 런타임 오류 없음.
- `768x1024`: 가이드 흐름 5/5 완료. `scrollWidth == clientWidth == 768`, 콘솔 런타임 오류 없음.
- `390x844`: 가이드 흐름 5/5 완료. `scrollWidth == clientWidth == 390`, 콘솔 런타임 오류 없음.
- 200% 확대 대체 검증: headless Chromium에서 실제 브라우저 확대 단축키는 적용되지 않고 CDP 배율은 Flutter WebGL 캔버스가 빈 화면으로 캡처되어, `1280px`의 200% 유효 폭에 해당하는 `640x450`으로 대체 검증했습니다. `scrollWidth == clientWidth == 640`, 콘솔 런타임 오류 없음.
- 콘솔 경고: WebGL/GPU `ReadPixels` 성능 경고만 관찰되었습니다. 사용자 기능 오류나 페이지 예외는 없었습니다.
- 발견 결함: 없음.
