# DevPath 프로토타입 디자인 시스템

## 의도

프로토타입은 넓은 기능 쇼케이스보다 하나의 증거 중심 LCS 학습 루프를 우선합니다. 첫 화면은 제품의 핵심 신호를 곧바로 보여줘야 합니다. 백엔드 실습 실패가 맥락 기반 리뷰, 보정된 다음 과제, 멘토에게 공유할 수 있는 학습 기록으로 이어지는 흐름입니다.

## 토큰

- 기본색: `#1F6F68`은 DevPath 정체성, 선택 상태, 승인된 맥락에 사용합니다.
- 정보색: `#2D6CDF`는 비교 결과와 생성된 결과에 사용합니다.
- 주의색: `#B7791F`는 일반 AI의 한계와 복구 가능한 주의 상태에 사용합니다.
- 위험색: `#9D3A3A`는 실패한 테스트, 잘못된 테스트 데이터, 오류 상태에 사용합니다.
- 배경색: `#F7F8F3`; 보조 표면색: `#E8EEE8`; 경계선: `#D9DED8`.
- 반경: 패널, 칩, 로고 마크, 안내 박스는 `8px`를 기본으로 합니다.
- 타이포그래피: 릴리스에서는 저장소에 포함된 한국어 폰트 자산을 우선합니다. 번들 폰트가 추가되기 전까지 실행 테마는 플랫폼 기본 대체 폰트를 쓰고, 테스트가 한국어 렌더링 가독성을 지킵니다.

## 컴포넌트

- 단계 본문에는 전체 너비 패널을 사용하고, 반복 카드가 필요한 경우에만 개별 증거 또는 맥락 항목을 카드로 표현합니다.
- 각 단계의 기본 동작은 가이드 루프를 앞으로 진행합니다. 재시작은 보조 동작입니다.
- LCS 필드 포함 여부는 개인정보와 직결되는 이진 선택이므로 스위치를 사용합니다.
- 내비게이션과 기본 CTA에는 아이콘을 사용하고, 설명 문구는 콘텐츠 패널 안에 둡니다.

## 반응형 규칙

- `<600px`: 현재 단계 요약과 펼칠 수 있는 단계 목록을 보여주는 소형 모드입니다.
- `600-979px`: 중간 폭 모드입니다. 일반 AI와 LCS 리뷰 비교 단계만 2열을 사용할 수 있습니다.
- `>=980px`: 레일 내비게이션과 전체 단계 표시기를 가진 데스크톱 모드입니다.
- Flutter 글자 배율 `2.0`과 브라우저 확대 `200%`에서도 텍스트가 읽혀야 하며, 모든 동작은 마우스 올림 없이 도달 가능해야 합니다.

## 접근성

- 단계 변경은 간결한 실시간 메시지로 알립니다.
- 미래 단계는 비활성화하고 선행 조건 안내를 제공합니다.
- 상호작용 대상은 최소 `44x44`를 목표로 합니다.
- 움직임은 짧고 보조적이어야 하며, 움직임 줄이기 사용자가 의미 파악을 애니메이션에 의존하지 않아야 합니다.

## 증거 규칙

- 취업 준비 증거는 관찰 가능한 학습 기록입니다. 준비도 점수, 배지, 채용 가능성 예측으로 표현하지 않습니다.
- 멘토 요약과 클립보드 내용은 정확히 같아야 합니다.
- 비활성화된 LCS 필드 값은 맥락 리뷰, 경로 보정, 증거, 보이는 요약, 클립보드 내용에 포함되면 안 됩니다.
