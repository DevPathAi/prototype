## 영향받는 페이지와 경로

- `/prototype/` 개요 히어로: 네 개의 기능 타일 대신 증거 중심 미리보기가 표시되고 `데모 > 학습 루프`를 여는지 확인합니다.
- `/prototype/` 데모 구역: `학습 루프`가 기본 데모 목적지이고 `기능 탐색` 안에 기존 독립 시뮬레이터가 남아 있는지 확인합니다.
- `/prototype/` 가이드 학습 루프: 소형, 중간 폭, 넓은 폭 단계 배치, 맥락 변경 시 초기화, 완료 증거, 멘토 요약 복사, 재시작을 검증합니다.

## 검증할 핵심 상호작용

- 하위 결과가 이미 있는 상태에서 승인된 LCS 필드를 변경합니다. 이후 결과가 초기화되고 맥락 단계에 `새 결과 생성` 복구 동작 하나만 표시되는지 확인합니다.
- 너비 `599`, `600`, `979`, `980`에서 이동합니다. `600-979` 중간 폭 비교 단계만 2열을 사용하고 다른 단계는 단일 열을 유지해야 합니다.
- 키보드로 모든 단계를 이동합니다. 초점 순서는 화면 순서와 일치하고, 단계/초기화/복사/오류 결과는 각각 한 번만 안내되어야 합니다.
- 멘토 요약 복사를 성공과 실패 복구 흐름으로 검증합니다. 복사된 내용은 보이는 승인 맥락 요약과 정확히 같아야 합니다.

## 경계 사례

- 위젯 테스트의 글자 배율 `2.0`: 잘린 라벨, 숨은 동작, 겹침, 가로 넘침이 없어야 합니다.
- 릴리스 빌드의 브라우저 확대 `200%`: 콘텐츠가 읽히고 동작에 도달할 수 있으며 마우스 올림 전용 단서가 없어야 합니다.
- 폰트 자산 실패: 폰트 파일 누락, 라이선스 누락, 해시 변경, 한국어 테스트 데이터 글리프 렌더링 실패는 배포 전에 검증을 실패시켜야 합니다.
- 정적 자산 예산 실패: 폰트 바이트 상한 초과 또는 전체 웹 자산 +10% 초과는 CI를 실패시켜야 합니다. 전체 자산 +5% 초과는 경고입니다.

## 핵심 경로

- 히어로 CTA -> 가이드 루프 시작 -> 맥락 승인 -> 맥락 기반 비교 -> 경로 차이 -> 취업 준비 증거 -> 멘토 요약 복사
- 기존 시뮬레이터 경로: 가이드 루프 -> `기능 탐색` -> 각 기존 시뮬레이터 -> 가이드 루프로 복귀했을 때 마지막 유효 단계 복원
- 릴리스 전달: `flutter analyze`, `flutter test --coverage`, 기능 커버리지 검사기, 릴리스 빌드, JS/정적 자산 예산 검사, Pages HTTP 스모크, 릴리스 브라우저 200% 확대 QA
